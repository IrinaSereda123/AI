🎯 Задание

Добавить AI-помощнику возможность самостоятельно узнавать реальную погоду через интернет.

В результате AI сможет:

  • 🌤️ отвечать на вопрос "Какая погода в Минске?" с реальными данными
  • 🌍 работать с любыми городами мира
  • 🤔 сам понимать, когда нужно использовать погоду, а когда ответить как обычно

⚙️ Как это работает (очень важно понять!)

Обычный AI не знает погоду. Он обучен на старых данных.

Мы научим AI распознавать, когда его спрашивают о погоде, и вместо того чтобы отвечать самому — вызывать специальную функцию, которая сходит в интернет за реальными данными.

Это называется Function Calling (вызов функций).

📖 Схема работы:
  1. Пользователь пишет: "Какая погода в Минске?"
  2. AI получает вопрос и понимает: "Для ответа мне нужна погода"
  3. AI возвращает не ответ, а команду: "Вызови функцию get_weather с параметром city='Minsk'"
  4. Наш код видит эту команду, идёт на сайт Open-Meteo и получает реальную погоду
  5. Отдаём результат обратно AI, чтобы он красиво оформил ответ
  6. Пользователь получает: "В Минске сейчас +15°C, облачно"

📁 Шаг 1. Добавляем функцию получения погоды

В файл script.js нужно добавить функцию, которая будет ходить на Open-Meteo API (бесплатный сервис погоды, не требует ключа).

📄 Добавить в начало script.js (после объявления переменных)
1// ===== ФУНКЦИЯ ДЛЯ ПОЛУЧЕНИЯ РЕАЛЬНОЙ ПОГОДЫ ===== 2async function getRealWeather(city) { 3 try { 4 // ШАГ 1: Получаем координаты города (широту и долготу) 5 const geoUrl = `https://geocoding-api.open-meteo.com/v1/search?name=${encodeURIComponent(city)}&count=1&language=ru&format=json`; 6 const geoResponse = await fetch(geoUrl); 7 const geoData = await geoResponse.json(); 8 9 // Если город не найден 10 if (!geoData.results || geoData.results.length === 0) { 11 return `Город "${city}" не найден. Попробуйте написать на английском.`; 12 } 13 14 // Берём первый найденный город 15 const location = geoData.results[0]; 16 const lat = location.latitude; 17 const lon = location.longitude; 18 const cityName = location.name; 19 20 // ШАГ 2: Получаем погоду по координатам 21 const weatherUrl = `https://api.open-meteo.com/v1/forecast?latitude=${lat}&longitude=${lon}¤t_weather=true&timezone=auto`; 22 const weatherResponse = await fetch(weatherUrl); 23 const weatherData = await weatherResponse.json(); 24 25 // Достаём температуру и скорость ветра 26 const temp = weatherData.current_weather.temperature; 27 const wind = weatherData.current_weather.windspeed; 28 29 // Возвращаем отформатированную строку 30 return `🌡️ Температура: ${temp}°C, 💨 Ветер: ${wind} км/ч`; 31 32 } catch (error) { 33 console.error('Ошибка погоды:', error); 34 return "Не удалось получить данные о погоде. Проверьте интернет."; 35 } 36}
📖 Что делает эта функция:
  • Строка 5: Отправляет запрос к геокодеру Open-Meteo, чтобы найти координаты города.
  • Строка 10-12: Если город не найден — возвращаем понятную ошибку.
  • Строка 21: Запрашиваем текущую погоду по координатам.
  • Строка 26-27: Извлекаем температуру и скорость ветра.
  • Строка 30: Возвращаем красивую строку с данными.

🔧 Шаг 2. Описываем инструмент для AI

Теперь нужно объяснить AI, какая функция у него есть и когда её вызывать. Это делается через объект tools.

📄 Добавить после функции getRealWeather()
1// ===== ОПИСАНИЕ ИНСТРУМЕНТА ДЛЯ AI ===== 2const weatherTool = { 3 type: "function", 4 function: { 5 name: "get_weather", // Название функции 6 description: "Получает текущую погоду в указанном городе", 7 parameters: { 8 type: "object", 9 properties: { 10 city: { 11 type: "string", 12 description: "Название города на русском или английском" 13 } 14 }, 15 required: ["city"] // Город — обязательный параметр 16 } 17 } 18};
📖 Что означает каждый параметр:
  • type: "function" — говорит AI, что это функция.
  • name: "get_weather" — имя функции, которое будет использовать AI.
  • description — описание того, что делает функция. AI читает это описание, чтобы понять, когда её вызывать.
  • parameters — описывает, какие данные нужны для вызова.
  • city — параметр с типом "string" (текст).
  • required: ["city"] — говорит, что город обязателен.

📖 Пояснение каждой новой строки

📌 Строки 27-54: функция getRealWeather(city)
Сама ходит в интернет за погодой. Использует Open-Meteo API (бесплатно, без ключа). Сначала находит координаты города, потом погоду по координатам.
📌 Строки 56-73: объект weatherTool
Описывает инструмент для AI: название функции, её описание, какие параметры нужны. AI читает это описание и понимает, когда вызывать функцию.
📌 Строка 103: tools: [weatherTool]
Передаём AI список доступных инструментов. В квадратных скобках можно перечислить несколько.
📌 Строка 104: tool_choice: 'auto'
Позволяет AI самому решать, вызывать функцию или ответить обычным текстом.
📌 Строка 112: if (message.tool_calls && ...)
Проверяем, вызвал ли AI наш инструмент. Если да — внутри лежат параметры вызова.
📌 Строка 114: JSON.parse(toolCall.function.arguments)
Превращает строку с параметрами в объект JavaScript. Например, '{"city":"Minsk"}' превращается в {city: "Minsk"}.
📌 Строка 119: await getRealWeather(city)
Вызываем нашу функцию, которая идёт в интернет за реальной погодой.
📌 Строки 133-137: role: 'tool'
Отправляем результат функции обратно AI. role: 'tool' означает "это результат работы инструмента".
