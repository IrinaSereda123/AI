Цель: Научить AI самостоятельно узнавать реальную погоду через интернет.

Что нового: Используем Function Calling (инструменты) — AI сам решает, когда ему нужно вызвать функцию получения погоды.

Как это работает: Вы спрашиваете "Какая погода в Минске?" → AI понимает, что нужна погода → вызывает нашу функцию → функция идёт на open-meteo.com за реальными данными → AI красиво оформляет ответ.

📖 Что нового во 2-й работе?

🔹 tools: [ { type: "function", function: { name: "get_weather", ... } } ]
Объявляем инструмент (функцию), которую AI может вызывать. Мы говорим AI: "У тебя есть возможность узнавать погоду, пользуйся ей, когда нужно".
🔹 tool_choice: "auto"
Позволяет AI самому решать, вызывать функцию или ответить обычным текстом. "auto" = автоматически.
🔹 message.tool_calls
Если AI решил вызвать функцию, здесь будут параметры вызова (например, название города). Мы проверяем это поле.
🔹 role: "tool"
После того как наша функция получила реальные данные, мы отправляем их обратно AI с ролью "tool" — это значит "вот результат работы инструмента".
🔹 Open-Meteo API (https://api.open-meteo.com)
Бесплатный сервис погоды. Не требует ключа. Сначала получаем координаты города, потом погоду по координатам.
🔹 Схема параметров функции (parameters)
Описываем, какие данные нужны функции: city (город) — строка, обязательный параметр. AI из вопроса извлечёт название города.
