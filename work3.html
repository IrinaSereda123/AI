📚 Асинхронные запросы в JavaScript

Как программы общаются с серверами через интернет

📖 Часть 1. Синхронный и асинхронный код

Синхронный код — выполняется строчка за строчкой. Каждая следующая команда ждёт, пока завершится предыдущая.

📄 Пример синхронного кода
console.log('1'); console.log('2'); console.log('3'); // Вывод: 1, 2, 3 (по порядку)

Асинхронный код — позволяет выполнять долгие операции (например, ожидание ответа от сервера) "в фоне", не блокируя остальную работу.

🎭 Простая аналогия: Вы заказываете пиццу по телефону. Синхронно — вы стоите у телефона и ничего не делаете, пока пиццу везут. Асинхронно — вы позвонили, заказали и пошли смотреть фильм. Курьер позвонит, когда приедет.

📖 Часть 2. Promise — обещание

Promise (в переводе "обещание") — это специальный объект в JavaScript, который представляет результат асинхронной операции, который будет получен в будущем.

У Promise есть три состояния:

  • pending (ожидание) — операция ещё не завершена
  • fulfilled (выполнено успешно) — операция завершена, есть результат
  • rejected (отклонено) — операция завершена с ошибкой
📌 Запомните: fetch() возвращает Promise. Когда вы пишете await fetch(), вы говорите: "подожди, пока Promise перейдет в состояние fulfilled или rejected".
📄 Пример Promise
// fetch() возвращает Promise const promise = fetch('https://api.example.com'); console.log(promise); // Promise {<pending>} // Через некоторое время Promise "выполнится" или "отклонится"

📖 Часть 3. GET и POST — два типа запросов

📥 GET — получить данные

Что делает: Запрашивает данные с сервера.

Когда использовать: Получить список, загрузить картинку, узнать погоду.

Где данные: В адресной строке (видно всем).

Пример: GET /users?id=5

📤 POST — отправить данные

Что делает: Отправляет данные на сервер.

Когда использовать: Отправить вопрос AI, зарегистрироваться, опубликовать пост.

Где данные: В теле запроса (скрыты).

Пример: POST /users с телом {"name": "Анна"}

📖 Часть 4. fetch() — главная функция для запросов

fetch(url, options)
Что делает: Отправляет запрос к серверу и возвращает Promise.

Параметры:
  • url — куда отправляем запрос (адрес сервера)
  • options — настройки запроса (метод, заголовки, тело) — необязательный параметр
📄 GET-запрос (получить данные)
fetch('https://api.example.com/users') // GET запрос по умолчанию
📄 POST-запрос (отправить данные)
fetch('https://api.example.com/users', { method: 'POST', // Указываем метод POST headers: { 'Content-Type': 'application/json' }, // Говорим, что отправляем JSON body: JSON.stringify({ name: 'Анна' }) // Данные для отправки })

📖 Часть 5. async / await — как дождаться ответа

async и await
async — ставится перед функцией. Говорит: "внутри этой функции будет await".
await — ставится перед fetch(). Говорит: "подожди здесь, пока придёт ответ".
📄 Пример async/await
// Объявляем асинхронную функцию (слово async) async function getUsers() { // Ждём ответ от сервера (слово await) const response = await fetch('https://api.example.com/users'); // Ждём, пока ответ превратится в объект const users = await response.json(); console.log(users); }
📌 Важно: Без async нельзя использовать await. Без await вместо ответа вы получите Promise.

📖 Часть 6. response.json() — превращаем ответ в объект

response.json()
Что делает: Превращает ответ сервера из текста в JavaScript-объект.

Зачем: Сервер присылает данные в виде текста (формат JSON). .json() превращает этот текст в объект, с которым удобно работать.
📄 Пример response.json()
// Сервер прислал текст: '{"name": "Иван", "age": 30}' const data = await response.json(); // Теперь data — это объект: { name: "Иван", age: 30 } console.log(data.name); // Выведет: "Иван" console.log(data.age); // Выведет: 30

📖 Часть 7. try / catch — ловим ошибки

try / catch
try — блок кода, который может вызвать ошибку.
catch — блок, который выполнится, если в try произошла ошибка.
📄 Пример try/catch
try { // Пробуем отправить запрос const response = await fetch('https://api.example.com/data'); const data = await response.json(); console.log(data); } catch(error) { // Если ошибка (нет интернета, неверный адрес) — попадём сюда console.log('Что-то пошло не так:', error.message); }
📌 Зачем это нужно: Без try/catch при ошибке программа просто остановится. С try/catch мы можем показать пользователю понятное сообщение.

📖 Часть 10. Практический пример — GET-запрос (погода)

🌦️ Простой сайт для получения погоды через Open-Meteo

Скопируйте этот код в файл weather.html и откройте в браузере. Введите город на английском (например, Minsk) и нажмите кнопку.

📄 weather.html — полный рабочий пример
<!DOCTYPE html> <html> <head> <title>Узнать погоду</title> </head> <body> <input type="text" id="city" placeholder="Город (на английском)"> <button id="weatherBtn">Узнать погоду</button> <div id="result"></div> <script> const btn = document.getElementById('weatherBtn'); const cityInput = document.getElementById('city'); const resultDiv = document.getElementById('result'); async function getWeather() { const city = cityInput.value.trim(); if (city === '') return; resultDiv.innerHTML = 'Загрузка...'; try { // Шаг 1: координаты города const geoUrl = `https://geocoding-api.open-meteo.com/v1/search?name=${city}&count=1&format=json`; const geoResponse = await fetch(geoUrl); const geoData = await geoResponse.json(); if (!geoData.results) { resultDiv.innerHTML = 'Город не найден'; return; } const lat = geoData.results[0].latitude; const lon = geoData.results[0].longitude; const cityName = geoData.results[0].name; // Шаг 2: погода const weatherUrl = `https://api.open-meteo.com/v1/forecast?latitude=${lat}&longitude=${lon}¤t_weather=true`; const weatherResponse = await fetch(weatherUrl); const weatherData = await weatherResponse.json(); const temp = weatherData.current_weather.temperature; const wind = weatherData.current_weather.windspeed; resultDiv.innerHTML = `${cityName}: ${temp}°C, ветер ${wind} км/ч`; } catch(error) { resultDiv.innerHTML = 'Ошибка: ' + error.message; } } btn.addEventListener('click', getWeather); </script> </body> </html>
⚠️ Важно: API Open-Meteo работает только с латинскими названиями городов. Например: Minsk, Paris, London, Berlin.
